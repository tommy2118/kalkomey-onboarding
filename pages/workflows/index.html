Development Workflow

Standards and best practices for efficient development workflows at Kalkomey.

Contents

Trunk-Based Development

We follow a Trunk-Based Development model with the default branch set to master. Key practices include:

  • Commit frequently to master.
  • Use feature flags for incomplete features.
  • Avoid long-lived branches.

Git Commit Messages

We adhere to the Git Commit Message Guidelines. A well-structured commit message includes:

Short summary (50 characters or less)

Detailed explanation of what the commit does, why it's necessary,
and any relevant context or background.

Use imperative mood and keep the summary concise.

ClickUp Integration

  • Link ClickUp tasks to branches using task IDs.
  • Update task statuses in ClickUp as progress is made.
  • Use ClickUp comments for code review discussions.

CI/CD Practices

  • Use automated pipelines for builds, tests, and deployments.
  • Enforce test coverage thresholds before merging.
  • Use staging environments to verify changes before production.
