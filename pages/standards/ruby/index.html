Ruby Style Guidelines

Formatting

  • Use 2 spaces for indentation
  • Keep lines under 120 characters
  • Use spaces around operators

Naming

  • Use snake_case for methods and variables
  • Use CamelCase for classes and modules
  • Prefix boolean methods with question mark

Rails Best Practices

Controllers

  • Keep controllers skinny, move logic to service objects
  • Use strong parameters for mass assignment

Models

  • Use scopes for common queries
  • Keep validation logic in models

Example Service Object

class UserRegistrationService
  def initialize(params)
    @params = params
  end

  def call
    user = User.new(user_params)
    return user if user.save

    false
  end

  private

  attr_reader :params

  def user_params
    params.require(:user).permit(:email, :password)
  end
end

Testing Standards

RSpec Guidelines

  • Write descriptive test cases using RSpec's expressive syntax
  • Use factorybot for test data
  • Maintain high test coverage

Example Test

RSpec.describe User, type: :model do
  describe "validations" do
    it { should validate_presence_of(:email) }
    it { should validate_uniqueness_of(:email) }
  end

  describe "#full_name" do
    it "returns the concatenated first and last name" do
      user = build(:user, first_name: "John", last_name: "Doe")
      expect(user.full_name).to eq("John Doe")
    end
  end
end

Component Guidelines

Hotwire Stack

Hotwire is a set of tools for building modern web applications with Ruby on Rails.

Key components include:

  • Stimulus: A modest JavaScript framework for the HTML you already have
  • Turbo: Drive your web app with minimal JavaScript
  • Turbo Streams: Real-time updates over WebSockets

TailwindCSS

TailwindCSS is a utility-first CSS framework for rapidly building custom designs.

Key features include:

  • Responsive design utilities
  • Customizable color palette
